# Session 1 — 2026-03-31

## What We Built
- **Buddy** — tamagotchi-style local AI companion for Claude Code
- Phases 1-4 complete (Foundation, Session Awareness, Intelligence, MCP)

## Key Decisions
- Python + Textual for TUI (easiest for non-programmer to maintain)
- Half-block Unicode pixel art (▀▄) with Rich color markup for 8-bit sprites
- Deterministic gacha with reroll (seed-based or random)
- Flexible AI backend (Ollama/OpenAI-compatible/offline fallback)
- MCP server with 5 tools for Claude Code integration
- Event-driven session observer via JSONL file

## Bugs Found & Fixed
- **`_render` method collision** — named our render method `_render` which shadows Textual's internal method, causing `NoneType` crash. Fix: renamed to `refresh_sprite`
- **`push_screen_wait` in `on_mount`** — Textual requires this in a worker. Fix: switched to callback-based `push_screen`
- **Hatch screen layout overflow** — fixed width container clipped on smaller terminals. Fix: responsive width + vertical button stack
- **No idle animation** — reactive frame watcher didn't trigger when value unchanged; old AnimationController over-throttled. Fix: simplified to direct `set_interval` + `advance_frame`
- **Wrong species** — seed used Windows USERNAME env var, not GitHub username. Fix: added hatch screen with seed choice + reroll

## Lessons Learned
- Always check for Textual internal method name collisions before naming widget methods
- Textual `Static` widgets need non-empty initial content or layout breaks
- Test TUI apps in a separate terminal, not via Claude Code's `!` command
- Fixed-width containers don't work well — use responsive `1fr` with `max-width`
- User works across multiple machines — always maintain in-repo HANDOFF.md
