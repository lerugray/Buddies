# Session 2 — 2026-03-31 (Evening)

## What We Built
Transformed **Buddy** into **BUDDIES** — a full multi-buddy collection system with cosmetic hats, 9 new species, and Party screen management.

## Key Achievements

### Multi-Buddy Refactor (Steps 1-2)
- Removed single-buddy constraint (`CHECK (id = 1)`)
- Added `AUTOINCREMENT`, `is_active`, `hat`, `hats_owned` columns
- Idempotent migrations for backward compatibility
- New store methods: `get_active_buddy()`, `get_all_buddies()`, `set_active_buddy()`, `update_buddy_by_id()`
- BuddyState now tracks `buddy_id`, `hat`, and `hats_owned`

### Hat System (Steps 3-4)
- 4 cosmetic hats with half-block pixel art: crown, wizard, propeller, tinyduck
- Stat-based unlock system:
  - Crown: high DEBUGGING (level 5+)
  - Wizard: high WISDOM (level 5+)
  - Propeller: high CHAOS (level 5+)
  - Tinyduck: starter gift at hatch
- Hat rendering integrated into sprite system (prepend to sprite, shiny border applies to full stack)
- Hat display in stats panel and owned hat list

### UI Overhaul (Steps 5-6)
- **HatchScreen**: Added name input, 4-tuple dismiss (species, shiny, seed, name)
- **App.py**: Refactored hatch flow, added hat unlock checks, replaced StatusBar with Footer
- **New keybindings**: [r]=hatch new, [p]=party
- **Fixed**: on_unmount() now uses `update_buddy_by_id` so multi-buddy saves work correctly

### Party Screen (Step 8)
- New `buddy/src/buddy/screens/party.py` with full buddy management
- Features: switch buddy, cycle hats, hatch new
- Keybindings: [enter]=switch, [h]=hat, [+]=hatch new, [escape]=close
- Arrow key navigation

### Expanded Species (Steps 9)

- All 9 have sprite frames (simple but distinctive pixel art)
- Total: 25 species now available
